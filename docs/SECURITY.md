# Security Guide

## Threat Model

Project Brain MCP is designed to expose a limited, read-only + planning-write view of local software projects to ChatGPT Developer Mode. The primary threats are:

1. **Prompt injection** causing unauthorized file reads or writes
2. **Path traversal** escaping the allowed project directories
3. **Secret leakage** through file contents or responses
4. **Unauthorized access** via the public MCP endpoint

## Defense Layers

### 1. Filesystem Sandbox

Every file path is processed through multiple checks:

- **Normalization**: `filepath.Clean` removes `.` and redundant separators
- **Traversal detection**: `..` components are rejected
- **Root containment**: Final path must be within the configured root
- **Symlink resolution**: `filepath.EvalSymlinks` resolves and re-validates containment
- **Drive letter / UNC checks**: Prevent Windows-specific bypasses

### 2. Blocklist

The following are blocked from reading and writing:

- `.env`, `.env.*`, `.envrc`
- `.ssh/`, `.aws/`, `.gcp/`, `.azure/`, `.kube/`
- `id_rsa`, `id_ed25519`, `id_ecdsa`
- `*.pem`, `*.key`, `*.pfx`, `*.p12`
- `secrets.*`, `credentials.*`
- `node_modules/`, `dist/`, `build/`, `.git/`

### 3. Write Restrictions

In `planning_write` mode (default):

- Writes are only allowed under `<project>/.chatgpt/` or `<project>/.ai/`
- Source files cannot be modified
- Paths are generated by the server, not provided by the client
- Atomic writes via temp file + rename

### 4. Secret Redaction

Response content is scanned for patterns including:

- `OPENAI_API_KEY`, `GITHUB_TOKEN`, `CLOUDFLARE_API_TOKEN`
- `AWS_ACCESS_KEY_ID`, `AWS_SECRET_ACCESS_KEY`
- `Bearer <token>`, `sk-...`, `ghp_...`
- `PRIVATE KEY` blocks

### 5. Audit Logging

Every tool call is logged to `~/.project-brain/audit/audit.jsonl`:

```json
{"ts":"2026-05-13T14:02:00+03:00","tool":"read_project_file","project_id":"...","path":"package.json","decision":"allowed","bytes_returned":4231}
```

### 6. Rate Limiting

A per-tool rate limiter prevents abuse (100 requests/minute by default).

## Authentication Modes

### Development: `noauth_dev`

- No authentication checks
- Only for local testing
- Must be combined with `read_only` or `planning_write` mode
- **Risk:** Anyone with the public URL can access the server

### Production: OAuth (Implemented)

The server implements a minimal OAuth 2.1 Authorization Code flow with PKCE:

- **Protected Resource Metadata** at `/.well-known/oauth-protected-resource`
- **Authorization Server Metadata** at `/.well-known/oauth-authorization-server`
- **Authorize endpoint** at `/oauth/authorize` — serves an HTML approval page
- **Token endpoint** at `/oauth/token` — exchanges authorization codes for JWT access tokens
- **PKCE required** — ChatGPT (public client) must send `code_challenge` and `code_verifier`
- **Single-user design** — Only the owner with the pre-shared `owner_secret` can approve connections

### How the OAuth Flow Works

1. ChatGPT connects to `/mcp` without a token
2. Server returns `401` with `WWW-Authenticate: Bearer resource_metadata="..."`
3. ChatGPT fetches metadata, discovers authorization server
4. ChatGPT redirects the user to `/oauth/authorize`
5. Server renders an approval page with a signed, time-limited token
6. Owner clicks **Approve Access**
7. Server redirects back to ChatGPT with an authorization `code`
8. ChatGPT exchanges the `code` + PKCE verifier for an access token
9. ChatGPT sends subsequent requests with `Authorization: Bearer <token>`
10. Server validates the JWT signature, expiry, issuer, and audience

### OAuth Security Properties

- **No external IdP** required (Google/GitHub not needed)
- **No session database** — Authorization codes stored in-memory only
- **One-time codes** — Each authorization code can only be exchanged once
- **Short-lived tokens** — Default 60 minutes, no refresh tokens
- **HMAC-signed approval links** — Prevents CSRF and forgery of the approval form
- **Time-bound approvals** — Approval tokens expire after 5 minutes

## Cloudflare Tunnel Security

- Tunnel creates outbound connections; no inbound port required
- Public hostname should use HTTPS
- Optional: Cloudflare Access with JWT validation at origin
- ChatGPT cannot send custom API keys or service tokens

## Security Tests

Run the test suite which includes:

- Path traversal attempts (`../.ssh/id_rsa`)
- Sensitive file access (`.env`, `.pem`)
- Write escapes (attempting to write outside `.chatgpt/`)
- Secret redaction verification
- OAuth flow: invalid tokens, expired codes, PKCE mismatch, reuse of codes

## Reporting Issues

If you discover a security vulnerability, please do not open a public issue. Contact the owner directly.
